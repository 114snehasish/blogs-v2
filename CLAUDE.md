## Commands

```bash
pnpm dev          # start dev server (http://localhost:5173)
pnpm build        # TypeScript check + production build (output: dist/)
pnpm preview      # serve the production build locally
pnpm lint         # run ESLint
pnpm migrate <wordpress-url> [--force] [--slug=<slug>]  # scrape WP posts into assets/posts/
```

All package management uses **pnpm**. Never use npm or yarn.

To add a shadcn/ui component:
```bash
pnpm dlx shadcn@latest add <component-name>
```

## Architecture

This is a **fully static** React + Vite site. Blog content lives as local Markdown files in `assets/posts/` (generated by the migration tool). There is no backend, no SSR, and no CMS.

### Content Flow

```
WordPress site (REST API)
        │
        ▼ pnpm migrate <url>
scripts/migrate.ts  ← CLI scraper (WP REST API → Markdown)
        │
        ▼
assets/posts/{slug}.md       ← committed Markdown files (frontmatter + body)
public/blog-images/{slug}/   ← downloaded images (all WP sizes)
        │
        ▼ import.meta.glob (bundled at build time)
src/services/local.ts  ← reads local .md files; no runtime HTTP
        │
        ▼
src/lib/markdown.ts    ← parses frontmatter (gray-matter) + body
        │
        ▼
src/hooks/useBlogPosts.ts / useBlogPost.ts  ← synchronous React hooks
        │
        ▼
src/pages/BlogListPage.tsx / BlogPostPage.tsx
```

### Migration tool

```bash
# Migrate all posts from WordPress
pnpm migrate https://blogs.snehasish-chakraborty.com

# Re-scrape everything (overwrite existing files)
pnpm migrate https://blogs.snehasish-chakraborty.com --force

# Migrate a single post by slug
pnpm migrate https://blogs.snehasish-chakraborty.com --slug=my-post-slug
```

Script entry: `scripts/migrate.ts` (run via `tsx`)
Supporting libs in `scripts/lib/`: `wordpress.ts`, `html-to-md.ts`, `image-downloader.ts`, `frontmatter-gen.ts`

### Vite plugin (auto-sync on build)

`plugins/wordpress-sync.ts` — currently commented out in `vite.config.ts`.
To enable: uncomment the import + plugin registration and set env vars:
```bash
WP_SITE_URL=https://blogs.snehasish-chakraborty.com pnpm build
WP_FORCE_SYNC=true  # optional: re-scrape all posts
```

### Blog post frontmatter schema

Every `.md` file in `assets/posts/` has YAML frontmatter matching `src/types/blog.ts`:

```yaml
---
title: "Post Title"
date: "2025-01-15"
slug: "post-slug"
description: "Plain-text excerpt"
tags: ["tag1", "tag2"]
draft: true                     # optional — hidden from listing
wordpressId: 123                # original WP post ID
author:
  name: "Author Name"
  url: "https://..."
  avatar: "/blog-images/authors/author-slug.jpg"
coverImage: "/blog-images/slug/featured.jpg"
coverImageResolutions:
  thumbnail: "/blog-images/slug/featured-150x150.jpg"
  medium: "/blog-images/slug/featured-300x200.jpg"
  mediumLarge: "/blog-images/slug/featured-768x543.jpg"
  large: "/blog-images/slug/featured-1024x683.jpg"
  full: "/blog-images/slug/featured.jpg"
seo:
  ogTitle: "..."
  ogDescription: "..."
  ogImage: "/blog-images/slug/og.jpg"
  twitterCard: "summary_large_image"
  canonicalUrl: "https://original-url/slug/"
---
```

### Cover image usage

- **Blog list page** — uses `coverImageResolutions.medium` (falls back to `coverImage`)
- **Blog post page** — uses `coverImageResolutions.large` → `.full` → `coverImage` (priority order)

### Routing

`react-router-dom` with `BrowserRouter`:
- `/` → `BlogListPage` — sorted list of all published posts
- `/blog/:slug` → `BlogPostPage` — individual post rendered via `react-markdown`
- `/about` → `AboutPage`

When deploying as a static site (e.g. GitHub Pages, Netlify, Cloudflare Pages), configure the host to redirect all 404s to `index.html` so client-side routing works.

### Tech stack
- **Vite 7** + `@vitejs/plugin-react` + `@tailwindcss/vite` (Tailwind v4, CSS-based config — no `tailwind.config.js`)
- **shadcn/ui** — components live in `src/components/ui/`, add via CLI above
- **react-markdown** + `remark-gfm` + `rehype-highlight` — markdown rendering with GFM and syntax highlighting
- **gray-matter** — YAML frontmatter parsing (requires `buffer` polyfill in `src/main.tsx`)
- **turndown** + `turndown-plugin-gfm` — HTML → Markdown (migration script only)
- **tsx** — TypeScript script runner (migration script only)

### Path alias

`@/` maps to `src/`. Configured in both `vite.config.ts` and `tsconfig.app.json`.

### Buffer polyfill

`gray-matter` uses Node's `Buffer`. A polyfill is injected at the top of `src/main.tsx`:
```ts
import { Buffer } from 'buffer'
;(globalThis as unknown as Record<string, unknown>).Buffer = Buffer
```
Do not remove this — it will cause a blank page crash.

## Styling convention

**Any element with more than 2 Tailwind utility classes must be extracted into a named CSS class in a component-level CSS file using `@apply`.**

- Every component/page that needs custom classes gets its own `.css` file alongside it (e.g. `Header.tsx` → `Header.css`).
- Import it directly in the component: `import './Header.css'`
- Name classes semantically after their role, not their appearance (e.g. `.post-title`, not `.text-xl-bold`).
- Each component CSS file must start with `@reference "../index.css"` (adjust the relative path depth) so Tailwind v4 can resolve `@apply` classes.
- Exception: Tailwind variant-modifier classes like `group` and `peer` cannot be used inside `@apply` — keep those inline in JSX and apply the rest via CSS class.
